#  ساختاردهی سیستم‌های چندعاملی
این پروژه، پاسخ عملی به چالش‌های ساخت یک پلتفرم تولید ویدیوی هوشمند  است. ما بر دو مشکل اصلی تمرکز کردیم: **۱. حفظ ثبات کاراکتر** و **۲. کنترل هزینه‌ها.**

---

### ۱. خلاصه 


| چالش اصلی | راهکار عملی | دستاورد |
| :--- | :--- | :--- |
| **فراموشی کاراکتر** (Consistency) | استفاده از **حافظه مشترک** و **Agent ناظر** برای اجرای قوانین ثابت. | جلوگیری از اشتباهات داستان‌گویی در طول صحنه‌ها. |
| **هدر رفتن بودجه** (Cost) | پیاده‌سازی منطق **تأییدیه چندگانه** و یک **Agent مدیر مالی** برای کنترل هزینه‌های API. | تضمین کیفیت قبل از انجام کار گران. |
| **ناهماهنگی و سوءتفاهم** | تعریف **مسیر اجباری (Workflow)** با **LangGraph** و ارتباط Agentها با زبان **کدگونه (JSON)**. | تضمین ترتیب منطقی داستان و جلوگیری از ابهام.  |

---

### ۲. اثبات مفهوم

در شبیه‌سازی پایتون، منطق Agentها با موفقیت نشان داده شده است:

* **نتیجه تست:** سیستم توانست یک تناقض عمدی در داستان ("لباس قرمز" برای کاراکتری با "زره آبی") را تشخیص دهد و قبل از تولید نهایی، آن را اصلاح و ثبات را حفظ کند. این نشان می‌دهد که منطق تأییدیه سه‌گانه کار می‌کند.

### ۳. نحوه اجرای کد

کد PoC برای سادگی با پایتون اجرا می‌شود.

1.  کد را اجرا کنید:
    ```bash
    python poc_consistency_agentic.py
    ```
2.  خروجی کنسول را بررسی کنید تا ببینید چگونه Agent ناظر، **ثبات کاراکتر** را در صحنه دوم **اجبار (Enforce)** کرده است.

---

### ۴. مستندات کامل

لطفاً فایل **`report.md`** را برای تحلیل عمیق‌تر، جزئیات راه‌حل‌ها و جدول زمان‌بندی بررسی کنید.
