 پاسخ به تسک ارزیابی: ساختاردهی سیستم‌های Agentic (ربات‌های هوشمند)

این پروژه، پاسخ عملی به چالش‌های ساخت یک پلتفرم تولید ویدیوی هوشمند است. ما بر دو هدف اصلی تمرکز کردیم: **۱. حفظ ثبات کاراکتر** و **۲. کنترل هزینه‌ها و بودجه پروژه.**

---

### ۱. خلاصه راه‌حل‌ها و دستاوردهای کلیدی

در این پروژه، ما Agentها را مجبور به رعایت قوانین محکم کرده‌ایم:

* **حفظ ثبات کاراکتر:** برای جلوگیری از فراموشی جزئیات (مثل تغییر ناگهانی لباس)، یک **حافظه مشترک** برای کاراکترها تعریف کردیم و یک **Agent ناظر** مسئول اجرای این قوانین است.
* **کنترل هزینه‌ها:** برای جلوگیری از هدر رفتن پول روی خروجی‌های نامعتبر، منطق **تأییدیه چندگانه (Cost-Control)** را پیاده‌سازی کردیم. این یعنی Agent مدیر مالی تأیید نکند، کار گران انجام نمی‌شود.
* **هماهنگی و نظم:** برای تضمین ترتیب منطقی داستان، Agentها باید از یک **مسیر اجباری (Workflow)** که با ابزارهایی مثل **LangGraph** تعریف شده، پیروی کنند. 
* **ارتباط بدون ابهام:** Agentها با هم فقط با زبان **کدگونه (JSON)** حرف می‌زنند تا هیچ سوءتفاهمی در انتقال دستورات پیش نیاید.

---

### ۲. اثبات مفهوم 
در شبیه‌سازی پایتون، Agentها با موفقیت منطق طراحی شده را اجرا کردند:

* **نتیجه تست:** سیستم توانست یک تناقض عمدی در داستان ("لباس قرمز" برای کاراکتری که لباس ثابتش "زره آبی" است) را تشخیص دهد. **سیستم قبل از تولید نهایی، این اشتباه را اصلاح کرد و ثبات را حفظ نمود.** این نشان می‌دهد که منطق تأییدیه چندگانه برای کنترل کیفیت و صرفه‌جویی، کاملاً عملی است.

### ۳. نحوه اجرای کد

کد PoC برای سادگی با پایتون اجرا می‌شود.

1.  کد را اجرا کنید:
    ```bash
    python poc_consistency_agentic.py
    ```
